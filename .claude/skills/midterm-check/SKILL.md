# 中期检查准备 (Midterm Check Preparation)

根据《仪器与电子学院毕业设计阶段中期检查表》和《阶段检查评分标准》，执行中期检查准备工作。

## 检查阶段说明

**第1阶段中期检查（12月）：**
- 辅导次数记录：≥10次（10分）
- 记录本笔记页数：≥10页（10分）
- 进度评价：50分
- 前期文档规范：30分（任务书10+开题报告10+中期总结10）

**第2阶段中期检查（4月）：**
- 辅导次数记录：≥8次（10分）
- 记录本笔记页数：≥8页（10分）
- 进度评价：80分

## 执行任务

### 1. 检查佐证材料清单

请检查以下材料的完成情况：

**必备材料：**
- [ ] 毕业设计任务书
- [ ] 毕业设计记录本
- [ ] 开题报告
- [ ] 中期总结报告

**项目佐证材料：**
- [ ] 软件代码（apps/, tools/, scripts/）
- [ ] 仿真结果（artifacts/）
- [ ] 试验记录/报告（docs/thesis/）
- [ ] 仿真代码或模型（apps/yolov8_rknn_infer.py）

### 2. 统计项目进度

请执行以下命令统计代码和文档：

```bash
# 代码行数统计
find apps/ tools/ scripts/ -name "*.py" -o -name "*.sh" | xargs wc -l

# 测试覆盖率
pytest tests/unit -v --cov=apps --cov=tools --cov-report=term-missing

# Git 提交统计
git log --oneline --since="2024-10-01" | wc -l

# 文档统计
find docs/ -name "*.md" | wc -l
```

### 3. 进度评价对照

根据评分标准，本项目进度对照：

| 进度选项 | 本项目情况 | 状态 |
|---------|-----------|------|
| 方案论证 | 开题报告完成 | ✅ |
| 完成代码编写 | apps/ 模块完成 | ✅ |
| 完成调试 | 单元测试通过 | ✅ |
| 完成建模 | YOLO模型转换完成 | ✅ |
| 部分完成设计/仿真 | PC模拟验证完成 | ✅ |

### 4. 生成中期检查报告

请生成报告到 `docs/thesis/midterm_report_X.md`，包含：

1. **基本信息**
   - 学生姓名、学号、专业
   - 设计题目
   - 指导教师

2. **进度情况**
   - 已完成工作清单
   - 代码/文档统计
   - 与任务书计划对比

3. **佐证材料清单**
   - 代码文件列表
   - 仿真结果截图
   - 测试报告

4. **下一步计划**
   - 待完成工作
   - 时间安排

- 硬件：RK3588边缘设备
- 软件：Python推理、C++部署
- 仿真：PC端RKNN模拟

生成报告到：
