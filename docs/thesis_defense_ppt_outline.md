### 第三部分：核心技术（6页，4分钟）

#### 【第8页】模型选型与优化
- **模型对比表**：
  | 模型 | 参数量 | 模型大小 | mAP@0.5 | 推理速度 |
  |-----|-------|---------|---------|---------|
  | YOLOv5s | 7.2M | 14MB | 45.6% | 15ms |
  | YOLOv8n | 3.2M | 6.2MB | 58.2% | 10ms |
  | **YOLO11n** | **2.6M** | **4.7MB** | **61.57%** | **8.6ms** |
- **选择依据**：轻量 + 高效 + 精准
- **配图**：模型结构示意图

#### 【第9页】分辨率优化策略
- **问题发现**：
  - 640×640: Transpose算子33600元素 → **CPU回退**
  - 416×416: Transpose算子14196元素 → **全NPU执行**
- **优化效果对比表**：
  | 指标 | 640×640 | 416×416 | 提升 |
  |-----|---------|---------|------|
  | 推理延迟 | 12.4ms | 8.6ms | ↓31% |
  | 端到端 | 24.3ms | 15.1ms | ↓38% |
  | FPS | 41 | **66** | ↑61% |
- **配图**：NPU利用率对比柱状图

#### 【第10页】后处理参数调优
- **NMS瓶颈分析**：
  - conf=0.25: 候选框~8000个 → NMS耗时3135ms ❌
  - conf=0.5: 候选框~500个 → NMS耗时5.2ms ✅
- **优化效果**：**提速600倍**
- **配图**：候选框数量与NMS延迟关系曲线图
- **公式**：NMS复杂度 O(n²)

#### 【第11页】INT8量化流程
- **量化流程图**：
  ```
  FP32模型 (11MB)
      ↓
  [校准数据集] 300张COCO person图像
      ↓
  [量化算法] w8a8 (权重+激活INT8)
      ↓
  INT8模型 (4.7MB) ✅
  ```
- **精度损失分析**：
  - 平均绝对差异：<1%
  - 最大相对误差：<5%
- **配图**：量化前后输出对比热力图

#### 【第12页】模型转换工具链
- **三步转换代码示例**：
  ```bash
  # Step 1: PyTorch → ONNX
  python tools/export_yolov8_to_onnx.py \
    --weights yolo11n.pt --imgsz 416

  # Step 2: ONNX → RKNN
  python tools/convert_onnx_to_rknn.py \
    --onnx best.onnx --do-quant

  # Step 3: 验证
  python scripts/compare_onnx_rknn.py
  ```
- **输出文件**：
  - best.onnx (11MB, FP32)
  - best.rknn (4.7MB, INT8)

#### 【第13页】部署与自动化
- **一键部署脚本**：
  ```bash
  scripts/deploy/rk3588_run.sh \
    --runner python \
    --model artifacts/models/best.rknn
  ```
- **自动化命令**（Claude Code）：
  - /full-pipeline: 完整转换流水线
  - /model-validate: 精度验证
  - /performance-test: 性能测试

## PPT设计建议

### 视觉风格
- **主色调**：科技蓝 (#1E88E5) + 深灰 (#424242)
- **辅助色**：橙色 (#FF9800) 强调，绿色 (#4CAF50) 成功标记
- **字体**：
  - 中文：微软雅黑 / 思源黑体
  - 英文/数字：Arial / Roboto
  - 代码：Consolas / Fira Code
- **排版**：简洁、留白充足、对齐工整

### 图表类型
- **流程图**：技术路线、工作流程
- **柱状图**：性能对比、mAP对比
- **雷达图**：多维指标对比（功耗、性能、成本）
- **饼图**：测试覆盖率、完成度
- **折线图**：优化效果趋势
- **表格**：参数配置、硬件规格

### 动画建议
- **淡入淡出**：文字内容逐条出现
- **擦除动画**：流程图分步展示
- **强调动画**：关键数据闪烁或放大
- **避免**：过度复杂的切换效果

### 代码展示
- **语法高亮**：使用代码块样式
- **字号适中**：确保后排可见（≥18pt）
- **关键行标注**：用箭头或高亮标注重点

### 图片使用
- **高清无水印**：至少1080p分辨率
- **统一风格**：配色、滤镜保持一致
- **标注清晰**：图片上的文字说明
- **来源标注**：必要时注明图片来源
